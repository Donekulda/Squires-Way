# Stage 2: Overview

Identify the purpose of each folder and subfolder with a basic overview.

## Checklist

- [ ] Read skeleton/folder-tree.md to know the structure
- [ ] For each top-level folder: analyze contents and write purpose (1–2 sentences)
- [ ] For each subfolder: analyze contents and write purpose (1 sentence)
- [ ] Output to overview/folder-purposes.md
- [ ] Create overview/toc.yml

## Output Format

```markdown
# Folder Purposes

## FolderName
Brief purpose of this folder (1–2 sentences).

### SubfolderName
Purpose of subfolder (1 sentence).

### AnotherSubfolder
Purpose (1 sentence).

## NextFolder
...
```

## Analysis Guidelines

- Focus on *what* the folder contains and *why* it exists
- Use code file names, namespace patterns, and README if present
- Avoid implementation details; keep it high-level
- If a folder is empty or generated, note that
